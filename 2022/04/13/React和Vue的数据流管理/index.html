前端的数据流与数据绑定

这篇文章旨在理清React和Vue中数据绑定和数据流之间的关联和区别

首先说一下前端的架构模式

架构模式

其实像官方文档所说，vue的核心库只关心视图层，react也是“用于构建用户界面”，也就是他们只关心View层，而我们一般说的vue，react是将其与既有工具链和各种库整合后的产物，

Model层和View层是不变的：

Model层：原始数据模型的管理 View层：用户直接操作的界面

MVC，MVP，MVVM的区别在与M与V的联系的部分，

即C：Controller，P：Presenter，VM：ViewModel

MVC中，Controller将用户的操作反馈给Model，通知其进行数据更新，

出现的问题：View层与Model层耦合，复用性差。我点击一个按钮，更新了Model并将数据渲染为List；这是我再点击一个按钮，同样更新这份数据但是渲染为Table。这个时候，由于之前逻辑已经连成一块，我们不得不再写一套渲染代码，而且当出了问题之后，因为各个模块是耦合在一起的，也不能快速判断究竟是哪个模块出现的问题。

MVP： Presenter: 比起Controller，Presenter会调用View层提供的接口去渲染Model。Presenter层既能将页面操作告知Model进行数据更新，又能在数据更新时负责通知View进行更新视图，使View层与Model层解耦。但是开发者依旧需要在Presenter中同时兼顾Dom和Data

ViewModel: 比起MVP中View需要自己提供API，MVVM在VM中构建一组状态数据（state data），作为View状态的抽象。然后通过数据绑定（data binding）使VM中的状态数据（state data）与View中的显示状态（screen state）保持一致。这样，VM中的展示逻辑只需要修改对应的状态数据，就可以控制View的状态，从而避免在View上开发大量的接口。

网上的一个例子：现在用户下拉刷新一个页面，页面上出现10条新的新闻，新闻总数从10条变成20条。那么MVC、MVP、MVVM的处理:

  1. View获取下拉事件，通知Controller
  2. Controller向后台Model发起请求，请求内容为下拉刷新
  3. Model获得10条新闻数据，传递给Controller
  4. Controller拿到10条新闻数据，可能做一些数据处理，然后拿处理好的数据渲染View
    MVC: 拿到UI节点，渲染10条新闻
    MVP: 通过View提供的接口渲染10条新闻
    MVVM: 无需操作，只要VM的数据变化，通过数据绑定，View直接变化

如果把Vue看成一个MVVM的框架，那么它的：

Model层：接口层，原始数据模型

View层：视图层，即template中的html代码

VM层：基于一个html元素构建的Vue实例。将Model中的原始数据模型，构建成一份View层可以使用的视图模型。这个时候，Model层、View层完全解耦。开发者只需要专注业务逻辑以及ViewModel层与Model的交互逻辑

而React并不能说是MVVM架构，

mvvm中如果vm对象的属性发生改变，对应的View层会部分更新，而react没有纯粹的vm对象，它只有属性和状态，通过属性和状态去映射视图

单向数据绑定与双向数据绑定

数据绑定就是View层和Model层之前的映射关系

单向数据绑定：Model层的更新会触发View的更新，而View的更新不会触发Model的更新，它们的作用是单向的。

双向数据绑定：Model的更新会触发View的更新，View的更新也会触发Model的更新，它们的作用是相互的。

React采用的单向数据绑定:

当用户访问View时，通过触发Events进行交互，而在相应Event Handlers中，会触发对应的 Actions，而Actions 通过调用 setState 方法对View的State 进行更新，State更新后会触发View的重新渲染。

可以看出，在 React 中，View 层是不能直接修改 State，必须通过相应的 Actions 来进行操作。

单向数据流与双向数据流

单向数据流

React、Vue 都只是单向数据流，即遵循父组件可以向子组件传递props，但是子组件数据的修改不会修改父组件传来的props，子组件只能通过事件通知父组件进行数据更改(react中子组件使用回调函数props.event()，父组件接受event={this.handleEvent}，Vue中的this.$emit)，如图：

而所谓的双向数据流，就是指在子组件中可以直接更新父组件的数据。

单向数据流使得程序的易读性更高，更直观。
